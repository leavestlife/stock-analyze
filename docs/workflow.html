CAN SLIM Quant Scanner · Workflow

실사용 종목 분석기 제작 흐름

레퍼런스 화면을 기반으로 시작해서, 실제 데이터 연결, 점수 산식 정렬, 상세 팝업, 차트, 공시·뉴스·애널리스트 데이터까지 붙인 현재 프로젝트의 전체 작업 흐름입니다.

scan.workflow
FrontendVanilla JS
BackendNode API
DataFMP + Alpha + DART
Verify9 tests passing

목표

종목을 자동으로 훑어서 “좋은 종목인가”와 “지금 들어갈 자리인가”를 분리해 보여주는 스캐너.

핵심 점수

TotalScore는 품질·실적·모멘텀, EntryScore는 타점·과열·눌림·손절 구조를 봅니다.

현재 방향

레퍼런스와 동일하게 메인 표는 요약, 종목 클릭 시 상세 분석 팝업으로 깊게 들어가는 구조입니다.

01기획

레퍼런스 구조 분석

왼쪽 섹터, 메인 종목 요약 표, 종목 클릭 상세 팝업, CAN SLIM·기술·재무·인사이트 탭 구조를 정리했습니다.

완료
02레이아웃

대시보드와 상세 팝업 구현

메인 컬럼, 11개 미국 섹터 분류, 검색 결과 누적, 시그널 정렬, 왼쪽 요약 카드, 상세 탭 UI를 구성했습니다.

완료
03데이터

실데이터 소스 연결

FMP 키 기반 가격·재무·어닝·뉴스, Alpha 보조 데이터, DART 한국 공시, SEC 링크형 공시/내부자/기관 흐름을 연결했습니다.

진행 중
04산식

CAN SLIM·퀀트·수학 지표 정렬

캡처로 받은 원본 산식 기준에 맞춰 원점수, 정규화 점수, 가중치, 기여도를 표시하도록 맞췄습니다.

완료
05상세

팝업 데이터 신뢰도 보강

차트, 기술 지표, 재무 지표, 뉴스 감성, 애널리스트 목표가, SEC/DART 공시, 도움말 툴팁을 보강했습니다.

완료
06검증

테스트와 브라우저 확인

server-contract.test.mjs로 검색·상세 payload·ETF/주식 분류·백테스트 계약을 확인하고, 브라우저에서 UI를 점검했습니다.

완료

데이터 파이프라인

  • 검색 입력: 티커·회사명·별칭을 실제 종목으로 해석
  • 가격 캐시: FMP 우선, Alpha/Yahoo 계열 보조 흐름
  • 재무/어닝: FMP fundamentals, earnings, analyst target 활용
  • 한국 공시: DART 검색 링크와 최근 공시 목록 표시
  • 미국 공시: SEC 검색, 13F·Form 4·filings 형태로 노출

점수 계산 흐름

  • CAN SLIM: C, A, N, S, L, I, M 기준별 점수화
  • 퀀트: 가치·평균회귀·모멘텀·MTF·낙폭·스마트머니 등
  • 수학: 허스트, 칼만, 통계 Z-Score 등 보조 평가
  • EntryScore: V4 Hybrid 기준으로 과열과 눌림 타점 분리
  • 각 카드 토글: 의미 설명, 입력 데이터, 계산 과정 표시

메인 시그널 의미

초록

EntryScore 75 이상. 진입 타이밍 후보입니다.

노랑

EntryScore 40~74. 관심 후보지만 관망·매집 관찰 구간입니다.

빨강

EntryScore 40 미만. 추격 위험 또는 진입 부적합 구간입니다.

현재 실행 파일

  • start_app.cmd: 서버 실행용 단일 진입점
  • server.mjs: API, 데이터 캐시, 산식 payload
  • app.js: 검색, 표 렌더링, 상세 팝업, 차트
  • styles.css: 현재 다크 커맨드센터 디자인
  • index.html: 앱 첫 화면 구조

남은 고도화 후보

  • SEC 13F/Form 4를 링크형에서 더 깊은 파싱형으로 확장
  • 뉴스 감성의 한글 요약 품질과 출처 신뢰도 점수 보강
  • 백테스트 결과를 종목·섹터별로 저장하고 화면에서 비교
  • 데이터 누락/대체 계산 여부를 점수 옆에 더 명확히 표시

이 문서는 지금까지 만든 기능과 검증 흐름을 사람이 빠르게 이해하기 위한 작업 지도입니다. 실제 앱은 프로젝트 루트의 start_app.cmd로 실행합니다.
